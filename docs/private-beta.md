# Private Beta Plan

This document defines the minimum operating model for the Vexart private beta during the v0.9 cycle.

## Target

- **10 active beta users**
- mix of:
  - CLI tool builders
  - OSS maintainers
  - one or two commercial evaluators

## Entry Criteria For Beta Users

Invite users who:

- already use Kitty / Ghostty / WezTerm
- can give feedback on real workloads
- are willing to report regressions with screenshots / repros

## Founder Intake Template

Collect:

- name / handle
- project type
- terminal used
- OS used
- expected use case
- whether they are evaluating for commercial use

## Feedback Buckets

Every beta report should be tagged into one bucket:

- rendering parity
- layout / sizing
- interaction / focus
- performance
- docs / onboarding
- packaging / install
- API ergonomics

## Weekly Review Questions

- Which regressions block real use?
- Which docs gaps create support churn?
- Which features are requested repeatedly?
- Are any deferred Phase 5 CSS-parity items now important enough to pull forward?

## Exit Condition

The private beta leg of v0.9 is considered operational when:

- this plan is documented
- founder has an invite checklist
- feedback can be bucketed consistently
- the docs site and packaging flow are ready to hand to testers
